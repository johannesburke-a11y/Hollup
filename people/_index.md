# People

> **Important humans with active relationships.**

## Purpose

One file per person with:
- **Current truth:** role, relationship status, what's currently relevant
- **Context:** how this person thinks, what's important to know
- **Timeline:** important interactions, decisions, changes
- **Open loops:** what's pending, follow-ups

## File schema

```
people/
├── _index.md
└── <firstname-lastname>.md
```

> File name: lowercase, hyphens, no special characters (`mueller` instead of `müller`).

## When to create a new person?

- When the person appears **repeatedly** in the system.
- When there's an **active relationship** (client, partner, colleague, mentor).
- Don't create for: one-off contacts, fleeting acquaintances.

## Rules

- One person = **one main file** (split-brain ban, see `RULES.md §2`).
- Current truth first, timeline append-only.
