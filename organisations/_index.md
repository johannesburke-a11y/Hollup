# Organisations

> **Companies, clients, suppliers, partners.**

## Purpose

One file per organisation with:
- **Current truth:** what this organisation does, your relationship to it, key contacts
- **Context:** important background knowledge
- **Timeline:** key events, decisions, changes in the relationship

## File schema

```
organisations/
├── _index.md
└── <org-slug>.md
```

## When to create a new organisation?

- When the organisation appears **repeatedly** in your work.
- When there's an **active relationship** worth tracking.

## Rules

- One organisation = **one main file**.
- Link to relevant people in `people/`.
